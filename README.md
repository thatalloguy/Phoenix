## Coding Style
`Classes` are UpperCase
`Variables` are camelCase
`Private` variables are _camelCase
`Enums` are UpperCase and CAP_CASE

NO usage of the stl, for example: `std::unordered_map`, `std::vector`, `std::function`.
Is allowed: `std::string` but use cstr as much as possible.
